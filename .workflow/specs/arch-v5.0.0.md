# Architecture — v5.0.0 "Aurora"

## Goals

- Make Fedora KDE 44 the supported target while preserving Fedora 43 best-effort compatibility.
- Add a read-only readiness layer on top of the v4 Atlas diagnostic architecture.
- Keep GUI, CLI, and support-bundle consumers thin; core/services own detection logic.
- Split optional Web API and daemon RPM runtime dependencies from the base GUI/CLI package.

## Decisions

- Readiness aggregation lives in `core/diagnostics/fedora44_readiness.py`.
- Desktop probes live in `services/desktop/kde44.py`.
- DNF5 and repository probes live in `services/package/dnf5_health.py`.
- The GUI uses an Atlas dashboard card and focused dialog, not a permanent new sidebar tab.
- CLI exposure is `fedora44-readiness`, with `--advanced` and global `--json`.
- Support Bundle v3 is generated by `core/export/support_bundle_v3.py` and embedded into the existing support ZIP path.
- RPM subpackages own optional runtime dependencies:
  - `loofi-fedora-tweaks-api`
  - `loofi-fedora-tweaks-daemon`

## Constraints

- All readiness checks are read-only.
- Repair actions must stay in the Atlas action model with risk metadata, command preview, and rollback/revert hints.
- UI modules must not run subprocesses directly.
- Service/core modules must not import PyQt6.
